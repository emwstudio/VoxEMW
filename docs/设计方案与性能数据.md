# VoxEMW · 良子实时语音对话系统 — 设计方案与性能数据

> 视频素材版 · 2026-07-19 · 所有性能数字均为 pod 实测（RTX 3090），非官方宣传值

## 一句话介绍

用一张 24GB 消费级显卡，在云端同时跑语音识别、大模型、声音克隆三个模型，实现和"大胃袋良子"的实时语音闲聊——他听得懂河北口音，用良子的音色和口吻，劝你使劲吃。

## 系统架构

```
浏览器（LiveKit Agents Playground / Sandbox）
        ↕ WebRTC（LiveKit Cloud 转发）
Runpod Pod（1× RTX 3090 24GB，$0.22/h）
 └─ LiveKit Agent Worker（livekit-agents 1.6.6）
     ├─ STT  Qwen/Qwen3-ASR-1.7B-hf（transformers 本地推理）
     │        silero VAD 判停 + StreamAdapter 整段识别
     ├─ LLM  Qwen/Qwen3-8B-AWQ（vLLM 0.11.2 OpenAI 兼容服务）
     │        int4 量化、关闭 think 块、repetition_penalty 1.15
     └─ TTS  openbmb/VoxCPM2（2B 参数，本地推理）
              Ultimate Cloning 三参克隆 + generate_streaming 流式输出
```

- 全链路开源模型、Apache-2.0 可商用、单卡可复现
- 显存分布（实测）：vLLM 9.5GB + VoxCPM2 ~8GB + ASR-1.7B ~4GB ≈ **21.5GB / 24GB**
- 音色克隆素材：10.86 秒良子原声 + 逐字台词（Ultimate Cloning 模式：prompt_wav + prompt_text + reference_wav 三参齐传）

## 性能数据（3090 实测）

| 环节 | 延迟 | 说明 |
|---|---|---|
| VAD 判停 | ~0.4s | 你住口到系统判定说完 |
| STT 识别 | **0.88s** | Qwen3-ASR-1.7B，一句话 |
| LLM 首 token | ~0.3–0.5s | Qwen3-8B-AWQ int4 |
| TTS 首包出声 | **0.14s**（预热后） | VoxCPM2 流式首 chunk |
| TTS 生成速率 | **RTF 0.52** | 说 1 秒语音约需 0.52s 生成（官方 4090 数据为 0.3） |
| 端到端接话 | **~1.5–2.5s** | 你住口 → 良子开口 |
| 进房预热 | ~35–40s | 三个模型加载（仅每房间首次） |
| 稳定性 | 5 分钟+ 连续对话零报错 | 多轮 STT→LLM→TTS 全链路 |

## 模型选型实录（A/B 现场对决，视频亮点）

| 候选 | 结果 | 实测表现 |
|---|---|---|
| MiniCPM5-1B | ❌ 淘汰 | 复读、死循环、变通用助手腔，1B 立不住人设 |
| Qwen3-4B-2507 | ⚠️ 备选 | 人设中但靠背模板，不会即兴 |
| **Qwen3-8B-AWQ** | ✅ **擂主** | 经典语录照用还会现挂：「AI 克隆你声音，那得先学会怎么把馒头咽下去」 |
| Qwen3.5-4B / Gemma4-E4B | ❌ 装不下 | 新架构起步 ~16GB，与 TTS+ASR 无法同卡共存；3.5-4B 求安慰场景还答非所问 |
| ASR 0.6B → **1.7B** | ✅ 升级 | 更准（方言/含混发音）且更快（1.4s→0.88s） |

## 良子人设工程（huashu-nuwa 蒸馏）

- 6 维度网络调研（语录/对话/表达DNA/他者评价/决策/时间线），1600 行调研档案、49 条语录原文、29 个对话场景，全部标注来源与可信度
- 提炼产物：5 个心智模型（胃袋叙事、打赏即最高优先级、压力资源化、嘴硬式精神胜利法、弹幕即罗盘）+ 8 条决策启发式 + 13 条表达 DNA
- 查证打假：「焖子圣经」是网友二创（查无原文）、涿州捐菜归属存疑、黑料全部无实据——写入诚实边界防止 AI 乱编
- 压缩为 <1000 字语音人设提示词，经 7 轮真实对话反馈迭代（开场简洁化、捧场/走心双模式、养生腔禁令、三宝知识卡、防复读、防"你懂什么"开头泛滥）

## 工程攻坚（踩坑记录）

1. **开头颤音**：波形分析实锤 VoxCPM2 每段合成开头有 ~160ms 低能量抖动 → 持续能量起点检测 + 8ms 淡入切除
2. **显存 OOM**：三模型同住 24GB，vLLM 从 0.45 降到 0.35 util + 防碎片分配
3. **驱动适配**：宿主机 CUDA 12.9 驱动跑不动 cu13 轮子 → vllm 锁 0.11.2（最后 cu128 时代版本）+ 独立 venv（与 agent 的 transformers 5.x 隔离）
4. **音频解码崩溃**：transformers 5.x 默认 torchcodec 与 torch 2.9 不兼容 → soundfile 解码 numpy 直传模型
5. **打断保护**：流式生成器线程 stop-event，用户插话立即停生成
6. **MiniCPM5 think 块泄漏**：reasoning 模型思考内容会念出来 → chat_template_kwargs 关 think
